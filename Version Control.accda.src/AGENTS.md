# AGENTS.md - AI Agent Guide to MSAccess VCS Export Structure

This file is automatically generated by the MSAccess VCS (Version Control System) add-in to help AI agents understand the exported file structure of Microsoft Access databases.

## Overview

This directory contains exported source files from a Microsoft Access database. The MSAccess VCS add-in converts Access database objects (forms, reports, queries, modules, etc.) into text-based source files that can be version controlled using Git or other VCS systems.

## Directory Structure

The exported source files are organized into folders by object type. Below is a comprehensive guide to the structure:

### Root Level Files

- **`vcs-options.json`** - Configuration file containing export/import options for this project
- **`vcs-index.json`** - Index file tracking changes, versions, and file hashes for fast save operations
- **`dbs-properties.json`** - DAO database properties
- **`documents.json`** - Document container metadata
- **`project.json`** - File format and Remove Personal Info flag
- **`proj-properties.json`** - CurrentProject properties
- **`vbe-project.json`** - VBA project properties
- **`vbe-references.json`** - VBA project references
- **`hidden-attributes.json`** - Hidden flag attributes
- **`nav-pane-groups.json`** - Custom navigation pane groups
- **`db-connection.json`** - Database connection metadata (for priming ODBC cache)
- **`Export.log`** - Export operation log file
- **`Build.log`** - Build operation log file (created during import/build)

### Object Type Folders

#### `forms/`
Access forms are exported here. Each form typically has:
- **`[FormName].bas`** - Form object definition (layout, controls, properties)
- **`[FormName].cls`** - Code-behind class module (only if "Split Layout from VBA" option is enabled)
- **`[FormName].json`** - Print settings (if the form has custom printer settings)

#### `reports/`
Access reports are exported here. Similar structure to forms:
- **`[ReportName].bas`** - Report object definition
- **`[ReportName].cls`** - Code-behind class module (only if "Split Layout from VBA" option is enabled)
- **`[ReportName].json`** - Print settings (nearly always present for reports)

#### `modules/`
VBA code modules (both standard and class modules):
- **`[ModuleName].bas`** - Standard VBA modules
- **`[ClassName].cls`** - Class modules

#### `queries/`
Access queries (SQL-based database objects):
- **`[QueryName].bas`** - Query object definition (contains source code, link data, and attributes)
- **`[QueryName].sql`** - Raw SQL definition (easier to read and see changes)

#### `macros/`
Access macros (automation objects):
- **`[MacroName].bas`** - Macro source code and build information

#### `tables/`
Table data (only exported if configured in options):
- **`[TableName].txt`** - Tab-delimited text format (more human-readable)
- **`[TableName].xml`** - XML table data (better data preservation)

#### `tbldefs/`
Table structure definitions (columns, data types, properties):
- **`[TableName].sql`** - SQL CREATE TABLE statement (human-readable)
- **`[TableName].xml`** - XML table definition (most complete, includes all metadata)
- **`[TableName].json`** - Linked table connection information (for linked tables only)

#### `relations/`
Database relationships between tables:
- **`[Table1][Table2].json`** - JSON representation of a relationship

#### `images/`
Images stored in the database:
- **`[ImageName].[ext]`** - Image file (PNG, JPG, etc.)
- **`[ImageName].json`** - Image metadata (mouseover name, internal name, extension, handler class)

#### `themes/`
Database themes (visual appearance):
- **`[ThemeName].thmx`** - Self-contained theme file (actually a ZIP file)
- **`[ThemeName]/`** - Extracted theme folder (if "Extract Theme Files" option is enabled)
  - Contains XML files for theme components

#### `vbeforms/`
MSForms 2.0 Forms (traditional VBA user forms):
- **`[FormName].frm`** - Form definition stub
- **`[FormName].frx`** - Binary OLE blob content and structure
- **`[FormName].json`** - JSON representation of content and structure

#### `imexspecs/`
Legacy table-based import/export specifications:
- **`[SpecName].json`** - JSON representation of import/export specification

#### `savedspecs/`
XML-based import/export specifications:
- **`[SpecName].json`** - Metadata with embedded XML

#### `tdmacros/`
Table Data Macros (data validation and automation):
- **`[TableName].xml`** - Table data macro definition

#### `menus/`
Command Bars (menus and toolbars):
- **`[CommandBarName].json`** - JSON representation of command bar structure and controls
- **`[CommandBarName]/`** - Folder containing command bar images (if any)
  - Contains bitmap image files (`.bmp`) used by the command bar

#### `functions/` (ADP Projects Only)
SQL Functions for Access Data Projects (ADP):
- **`[FunctionName].sql`** - SQL function definition

#### `procedures/` (ADP Projects Only)
Stored Procedures for Access Data Projects (ADP):
- **`[ProcedureName].sql`** - Stored procedure definition

#### `views/` (ADP Projects Only)
Server Views for Access Data Projects (ADP):
- **`[ViewName].sql`** - SQL view definition

#### `sqltables/` (ADP Projects Only)
SQL Tables for Access Data Projects (ADP):
- **`[TableName].sql`** - SQL table definition

#### `triggers/` (ADP Projects Only)
SQL Triggers for Access Data Projects (ADP):
- **`[TriggerName].sql`** - SQL trigger definition

#### `databases/` (External Database Schemas)
External database schema exports (MySQL and SQL Server):
- **`[DatabaseName]/`** - Subfolder for each configured external database connection
  - Contains exported schema objects from the external database
  - Structure varies by database type (MySQL vs SQL Server)
  - May include subfolders for tables, views, functions, procedures, triggers, etc.
  - **Note**: This is an export-only feature for tracking schema changes

## File Naming Conventions

- Object names in Access can contain spaces and special characters
- File names use the exact object name from Access
- Extensions indicate the file type:
  - `.bas` - Basic/VBA source code
  - `.cls` - Class module
  - `.sql` - SQL statements
  - `.json` - JSON metadata
  - `.xml` - XML definitions
  - `.txt` - Tab-delimited text
  - `.thmx` - Theme file (ZIP format)
  - `.frm` / `.frx` - VBA form files
  - `.bmp` - Bitmap image files (used by command bars)

## Key Concepts

### Fast Save
The add-in uses a "fast save" feature that only exports objects that have changed since the last export. This is tracked in `vcs-index.json` using file hashes.

### Split Layout from VBA
When enabled, forms and reports export their layout (`.bas`) separately from their code-behind (`.cls`). This makes it easier to see changes in either the UI or the code.

### Sanitization
Exported files are sanitized to remove noise (machine-specific differences, timestamps, etc.) that don't affect functionality but create unnecessary diffs in version control.

### Encoding
Files are exported as UTF-8 with BOM (Byte Order Mark) to ensure proper handling of international characters. The UTF-8 BOM (EF BB BF) is critical for proper import back into Access. **AI agents must preserve the original UTF-8 BOM encoding when modifying any text files.** Changing the encoding or removing the BOM will cause import failures.

## Working with These Files

### Reading Code
- VBA code is in `.bas` and `.cls` files
- SQL queries are in `.sql` files
- These are standard text files that can be edited in any text editor

### Understanding Structure
- JSON files contain metadata and configuration
- XML files contain complete object definitions
- SQL files contain human-readable table/query definitions

### Making Changes
1. Edit the source files directly
2. Use the VCS add-in to build/import the changes back into Access
3. The add-in will merge changes and handle conflicts

## Important Notes for AI Agents

1. **CRITICAL: Preserve UTF-8 BOM Encoding** - All text files are exported with UTF-8 BOM encoding (EF BB BF bytes at the start). **You MUST preserve this encoding when modifying files.** Do NOT:
   - Change encoding to UTF-8 without BOM
   - Change encoding to ASCII, ANSI, or any other encoding
   - Remove the BOM from existing files
   - Add BOM to files that don't have it (check first)
   - Changing encoding will cause import/build failures. Always save modified files with UTF-8 BOM encoding.

2. **Don't modify `vcs-index.json` manually** - This is managed by the add-in

3. **Be careful with XML files** - They contain complete object definitions and must remain valid XML

4. **JSON files are configuration** - Modify with care, ensure valid JSON syntax

5. **File paths are relative** - All paths in this structure are relative to the export folder root

6. **Not all folders may exist** - Only folders for objects actually in the database will be present

7. **Binary files** - `.frx` files are binary and should not be edited as text

8. **ADP Projects** - Access Data Projects (`.adp` files) may include additional folders for SQL Server objects (`functions/`, `procedures/`, `views/`, `sqltables/`, `triggers/`)

9. **External Database Schemas** - The `databases/` folder contains export-only schema information from external databases (MySQL/SQL Server). These are for tracking purposes and are not imported back into the external database

10. **Command Bars** - Legacy menus and toolbars are exported but may have limited import support for custom built-in command bars
